1、关于配置DBGp Proxy选项的 需要下载 DBGP proxy软件并运行的 目的是解决多个人同时的调试，如果是一个人本地调试就不用管它.：是的

2、似乎本地调试和远程调试都没有必要安装Xdebug Helper。目前发现是这样。

3、 环境：windows + docker+xdebug

3.1 配置 php->Dockerfile->添加 xdebug的配置。
执行 # docker-compose up --build
3.2 php.ini   (php 7.3.33 - xdebug :3.1.6)
追加

[Xdebug]
xdebug.mode=debug
xdebug.idekey=myidkey 
xdebug.start_with_request=yes
;xdebug.discover_client_host=true告诉 Xdebug 尝试从 HTTP 请求中提取客户端的 IP,而xdebug.client_host告诉它如果这不起作用,应该尝试什么。
xdebug.discover_client_host=true
xdebug.client_host=host.docker.internal
xdebug.client_port=9013
