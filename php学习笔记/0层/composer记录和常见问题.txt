常见问题
1、Cannot create cache directory /home/ubuntu/.composer/cache/repo/https---mirrors.aliyun.com-composer/, or directory is not writable. Proceeding without cache

sudo chown -R ubuntu /home/ubuntu/.composer/
sudo chown -R ubuntu /home/ubuntu/.composer/cache/repo/https---mirrors.aliyun.com-composer
sudo chown -R ubuntu /home/ubuntu/.composer/cache/files/

2、composer file_put_contents(./composer.json): failed to op
sudo chown -R ubuntu:ubuntu  ./composer.lock

3、undefine XX 
是插件的文件，通过 加 -vvv 定位到对应的文件看一下就行

4、GPG error: The following signatures couldn't be verified because the public key is not available

sudo gpg --keyserver keyserver.ubuntu.com --recv 5523BAEEB01FA116 //(这个公钥根据提示来写的)
sudo gpg --export --armor 5523BAEEB01FA116 | sudo apt-key add -
sudo apt-get update

7、Install of composer/package-versions-deprecated failed
解决： composer install --no-plugins

8、requires psr/log, it could not be found in any version, there may be a typo in the package name
解决： 包的来源问题
composer config -g repo.packagist composer https://packagist.org

composer require 参数说明：
申明依赖-参数
--prefer-source: 当有可用的包时，从 source 安装。
--prefer-dist: 当有可用的包时，从 dist 安装。
--dev: 安装 require-dev 字段中列出的包。
--no-update: 禁用依赖关系的自动更新。
--no-progress: 移除进度信息，这可以避免一些不处理换行的终端或脚本出现混乱的显示。
--update-with-dependencies 一并更新新装包的依赖。
